CMS Architecture

Static Site Generation

  • Source format: Markdown files with YAML front matter
  • Output format: Static HTML pages
  • Template system: Shared layouts and components
  • Automation: GitHub Actions for content synchronization

File Structure

content/
├── *.md                    # Source blog posts
├── blog-index.json         # Generated post index
└── posts/
    └── [slug]/
        └── index.html      # Generated post pages

Blog Post Creation Workflow

1. Post Generation

npm run generate-post -- \
  --title="Your Post Title" \
  --description="Post summary" \
  --category="Web Development" \
  --tags="html,css,accessibility"

2. Front Matter Structure

---
title: "Post Title"
description: "SEO-friendly description"
publishDate: "2025-01-15"
category: "Web Development"
tags: ["html", "css", "accessibility"]
featuredImage: "/images/post-featured.svg"
author: "Anthony Cassella"
readingTime: "5 min read"
---

3. Content Authoring

  • Markdown syntax: Standard Markdown with extensions
  • Code highlighting: Syntax highlighting for code blocks
  • Image optimization: Responsive image handling
  • Internal linking: Cross-reference between posts

4. Content Synchronization

npm run sync-cms

Process:

  1. Parse Markdown files with front matter
  1. Generate HTML from Markdown content
  1. Apply shared template with navigation/footer
  1. Update blog index with post metadata
  1. Create individual post directories

Client-Side Loading

// blog-dynamic-load.js
class BlogLoader {
  async loadPosts(filters = {}) {
    const response = await fetch('/content/blog-index.json');
    const data = await response.json();
    
    let posts = data.posts;
    
    // Apply filters
    if (filters.category) {
      posts = posts.filter(p => p.category === filters.category);
    }
    
    if (filters.tag) {
      posts = posts.filter(p => p.tags.includes(filters.tag));
    }
    
    return posts;
  }
  
  renderPosts(posts) {
    // Generate HTML for post cards
  }
}

Performance Considerations

Static Generation Benefits

  • Fast loading: Pre-generated HTML
  • CDN friendly: Cacheable static assets
  • SEO optimized: Server-side rendering
  • Secure: No dynamic server vulnerabilities

Optimization Techniques

  • Image compression: Automated image optimization
  • Code splitting: Minimal JavaScript per page
  • CSS optimization: Critical CSS inlining
  • Lazy loading: Non-critical content deferred
